If you just want an LED to flicker, the simplest solution is don't use the LCD signals at all. Just connect an LED (with a resistor) to a spare GPIO and toggle it in software:

C
while (1) {
    gpio_put(LED_PIN, 1);
    sleep_ms(100);
    gpio_put(LED_PIN, 0);
    sleep_ms(100);
}

If you specifically want the LED to flicker from an LCD signal without extra code, connect it (through a resistor) to the E (Enable) pin. It will flash whenever data is sent to the LCD, but the pulses are so brief that the LED may not be visibly bright unless the display is being updated continuously.

For a visibly flickering LED, a dedicated GPIO is the better approach.

do i have enough pins on the pico to run a tft

ChatGPT said:

Yes, a Raspberry Pi Pico has plenty of GPIOs for a TFT display in most projects.

For a typical SPI TFT (such as an ST7735, ST7789, ILI9341), you'll usually need:

  • SCK (SPI clock)

  • MOSI (SPI data)

  • CS (chip select)

  • DC (data/command)

  • RST (reset, sometimes optional)

  • BL (backlight, optional)

That's typically 5–6 GPIOs.

The Pico has over 20 usable GPIOs, and SPI pins can be mapped to multiple GPIOs.

For example:

FunctionGPIO
SCKGP18
MOSIGP19
CSGP17
DCGP16
RSTGP20
BLGP21

That's only 6 pins.

If you're building a robot or controller, tell me:

  • Which TFT module (ST7735, ST7789, ILI9341, etc.)?

  • What else is connected to the Pico (encoders, motors, ESCs, buttons, sensors)?

I can help estimate the total pin usage and check whether you'll run out.
